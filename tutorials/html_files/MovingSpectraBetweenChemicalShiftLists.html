Moving Spectra between Chemical Shift Lists

Background

Each spectrum in your project always has to be associated with a Chemical Shift List. When you open a new spectrum, it will by default be associated with the first Chemical Shift List in your project (usually the default list). The important thing to remember is that the Chemical Shifts are always averaged across the peak positions of all Peaks in all Spectra belonging to that Chemical Shift List (unless the Peak's Figure of Merit is set to 0.0). For more detailed information about how Chemical Shifts and Chemical Shifts Lists are handled in CcpNmr Analysis, have a look at our Chemical Shifts page and video.

You should, therefore, use different Chemical Shift Lists for each group of spectra recorded under different conditions resulting in (potentially) changed chemical shifts. For example, if you have spectra which were recorded on different constructs or mutants, they should be associated with different Chemical Shift List. Similarly, for a titration, each of your spectra should (ideally) be in a separate Chemical Shift List. So if you record some spectra at a different temperature, a different pH or including different concentrations of a ligand or binding partner, then you should use a different Chemical Shift List. Otherwise, your Chemical shifts will be averaged across all the spectra in the titration. If you did not put them in different Chemical Shift Lists, the chemical shifts of the peaks that move will contribute towards the chemical shift of that NmrAtom. And when you go to deposit your chemical shifts in the BMRB, some of them might be distorted!

If you wanted, you could also create a Chemical Shift List that you use specifically to collect/dump all the Chemical Shifts of Spectra which you don't want to have contributing to your main Chemical Shift list for deposition. This could be for dynamics spectra, RDC spectra, Titrations etc. - anything that shouldn't contribute towards the Chemical Shifts you want to deposit in the BMRB.

Moving Spectra manually

If you double-click on a spectrum in the sidebar to bring up the Spectrum Properties dialog box, you can see in the General tab which Chemical Shift List the Spectrum is associated with. You can also change which Chemical Shift List this is (including creating a new Chemical Shift List) using the drop-down menu.

An alternative way to move Spectra between Chemical Shift Lists, is to double-click on a Chemical Shift List in the sidebar. This will show you which spectra are associated with this Chemical Shift List and which are not. You can drag spectra between the Not Included and Included boxes to change which spectra belong to that Chemical Shift List (or double-click on a spectrum to move it to the other box). If you only have one Chemical Shift List in your project, you will not be able to exclude spectra from this. If you move spectra out of the current Chemical Shift List, then they will by default go into the first Chemical Shift List. You can't move spectra out of the top Chemical Shift List - move them into another one instead.

Moving Spectra automatically using a macro

In the Macro / Run CCPN Macros menu you will find a macro called moveSGSpectaIntoNewCLs. This will allow you to move all the Spectra in a SpectrumGroup into new Chemical Shift Lists in one go. Running the macro will bring up a little pop-up. There are several options once you have selected the SpectrumGroup you wish to use. You can move your Spectra into

  • separate new Chemical Shift Lists: this is suitable if you have a titration in which each Spectrum in your SpectrumGroup should be a separate Chemical Shift List. The Chemical Shift Lists will simply be named according to the spectra.
  • same new Chemical Shift List: this will allow you to move all the Spectra in the selected SpectrumGroup in a new ChemicalShiftList whose name you can set yourself. This will be useful if you have spectra recorded on a new construct, mutant, complex or protein.
  • existing Chemical Shift List: this will allow you to move all the Spectra in a selected SpectrumGroup into an existing Chemical Shift List which you can select using a drop-down menu. Again, this will be useful if you have spectra recorded on a new construct, mutant, complex or protein, but enables you to add any Spectra that you hav enewly added to the SpectrumGroup to be added to the correct Chemical Shift List as well.
